## 七、写作风格指南

### 7.1 语气与人称

- 使用 **第一人称**（"我"、"本实验"、"本次作业"）
- 正式学术中文，避免口语化，但保留工程实践感的表述
- 以 **"在本次《嵌入式系统》综合作业中，我基于……"** 风格开头
- 段落间逻辑连贯，采用"首先……其次……最后……"等过渡

### 7.2 各章节写作模板

#### 一、实验目的
- 以概括性语句开头（"本次XX作业/实验旨在……"）
- 使用 **"具体达成以下学习目标："** 引出目标列表
- 每条目标以动词短语开头：**掌握……、理解……、学会……、提升……**
- 每个目标后跟冒号，用一句话深入解释该目标的具体内涵

#### 二、实验环境
- 以铺垫句开头（"本实验通过……完成了……具体实验环境如下："）
- 分 4 个子项：
  1. **硬件平台与测试环境** — 主控芯片型号+架构参数、开发板型号、供电方式、下载调试工具、通信监测工具
  2. **外设模块与元件清单** — 面包板+杜邦线+各模块型号规格
  3. **引脚分配与连接关系** — IO口 → 外设 → 复用功能的映射表，附共地声明
  4. **软件平台与配置** — IDE、编译器、固件库、时钟/NVIC 等核心参数

#### 三、实验内容
- **3.1 功能描述**：整体工作流程概述 + 指令集表格
- **3.2 电路原理图**：硬件连接原理文字详述 + **插入硬件原理图图片**（用户提供路径）
- **3.3 软件实现原理**：程序逻辑分段解析（初始化流程 → 主循环逻辑 → 中断服务逻辑 → 状态机设计），需插入**代码流程图图片**（用户提供路径）
- **3.4 关键代码解析**：选取 4–6 个核心代码片段，每个按三段式结构（标题+代码+解析），代码需完整语法高亮
- **3.5 现象与应用场景**：典型测试现象描述 + 实际应用场景探讨

#### 四、实验总结
- 第一段：总览技术架构回顾（"回顾整体技术架构，本系统以……"）
- 第二段：遇到的工程问题及解决过程（"在代码编写与实物调试的过程中，我遇到了……"）
- 第三段：核心收获与能力提升（"本次作业的核心收获在于……"）
- 第四段：扩展功能亮点阐述（如有，如 CALC 递归下降法）
- 第五段：未来改进方向（"尽管基本达成了课程要求，但本系统仍有……我计划……"）

#### 五、完整代码（附录）
- 用户指定方案 **5B**：仅附关键代码文件（如 `main.c`），非全文粘贴
- 代码字体 **Consolas 8pt**，单倍行距，**保留语法高亮**
- 代码之前用黑体 14pt 加粗标题 "五、完整代码"

### Q4：实验内容
- **功能描述**（本实验要实现什么？支持哪些指令/功能？）
本实验是一个纯中断驱动的系统（主循环 while(1) 为空），主要演示不同抢占优先级中断之间的相互打断与嵌套现象。它支持以下核心功能与指令动作：

基础心跳指示（底层平民）：

利用 SysTick（系统滴答定时器），每 500ms 翻转一次 LED0 (PB4)，作为系统正常运行的“心跳”。

按键多态检测与系统阻塞（拦路虎）：

按下 KEY1 (PA0)：

双击：400ms内连续按下两次，翻转 LED2 (PB6) 的亮灭状态。

单击：若400ms超时未触发第二次按键，系统判定为单击，点亮 LED1 (PB5) 并故意死机（死延时）3秒。此时，比它优先级低的“心跳” LED0 将停止闪烁（心跳骤停）。

旋钮调光（绝对特权）：

旋转编码器 (PA1, PA2)：随时可以顺时针或逆时针旋转，步进调节 LED3 (PB7) 的 PWM 亮度。

特权表现：即使系统正处于按键单击触发的“3秒死机”状态中（心跳已停），旋转编码器依然可以无缝打断阻塞，实时调节灯光亮度，演示了最高抢占优先级对中等优先级的绝对压制。
- **关键技术点**（用到了哪些外设？哪些协议？哪些算法？）
本实验虽然代码量不大，但融合了 STM32 的多项核心机制：

核心外设：

NVIC (嵌套向量中断控制器)：全局配置 NVIC_PriorityGroup_4（全抢占优先级，无响应优先级），用于设定严格的打断层级。

EXTI (外部中断/事件控制器)：配置上升沿（编码器）和下降沿（按键）触发。

SysTick (系统滴答定时器)：用于生成周期性心跳中断，且其默认极高的优先级被强制降级。

TIM4 (通用定时器)：使用 TIM_OCMode_PWM1 模式输出 PWM 波形实现 LED 调光。

核心算法与逻辑：

纯软件精准延时 (Delay_ms)：通过双重 for 循环实现的死延时。关键点：抛弃了常规的 SysTick 延时，防止在高级中断中调用时因 SysTick 被屏蔽而导致系统彻底死锁。

中断内的状态机/超时轮询：在 EXTI0 中断服务函数内部，利用纯软件循环检测按键的第二下触发，实现“单击/双击”识别。

正交编码器方向解码：在A相触发中断时，通过读取B相的电平状态（0或1）来判断旋转方向（顺/逆时针）。

按键去抖（Debouncing）：在中断内利用软件延时跳过机械抖动期。
- **需要解析的核心代码段**（列出 4–6 个要详细讲解的代码片段主题）
NVIC 抢占优先级分组与分配配置

位置：main() 函数中的 NVIC_PriorityGroupConfig 及各种 NVIC_Init。

解析重点：解释 Group 4 的含义（4位全用于抢占）。重点说明 1级（特权）、2级（拦路虎）、3级（底层心跳）的抢占逻辑，以及“数字越小优先级越高”的 Cortex-M 规则。

强制降低 SysTick 优先级的核心操作

位置：main() 函数中的 NVIC_SetPriority(SysTick_IRQn, 3);。

解析重点：解释 SysTick 属于内核异常，默认优先级极高。如果不显式降级为 3，它将无法被外部中断（EXTI）打断，整个“心跳骤停”的实验逻辑就不成立。

防死锁的纯软件延时函数设计

位置：Delay_ms() 函数实现。

解析重点：为何不用常见的 delay_ms()（通常依赖 SysTick 计数）？深入探讨中断上下文（Interrupt Context）中调用依赖低优先级中断的阻塞函数会导致死锁（Deadlock）的致命危害。

在中断中实现“单击/双击”检测的非主流写法

位置：EXTI0_IRQHandler() 中的 while(wait_cnt < 400) 轮询模块。

解析重点：分析这段代码如何通过“阻塞等待 400ms”来区分单击和双击。探讨这种写法的教学意义（为了故意霸占 CPU 资源），并补充说明在实际工程开发中应当极力避免此类写法（应改用状态机+定时器中断）。

旋转编码器的极简正交解码逻辑

位置：EXTI1_IRQHandler() 内部方向判断。

解析重点：结合编码器 A/B 相的波形时序图，解释为什么“在 A 线上升沿时，直接读 B 线电平”就能 100% 准确判断出正反转。
